# Contributing to Metaverse Rang – Sovereign Assets NFT VOD

First of all, thank you for considering contributing to Metaverse Rang.

This project aims to build an open ecosystem for sovereign digital assets, NFT-powered ownership, metaverse experiences, and decentralized video-on-demand (VOD) infrastructure. Every contribution, no matter how small, helps improve the project and strengthen the community.

---

## Ways to Contribute

There are many ways you can contribute:

- Reporting bugs and issues
- Suggesting new features
- Improving documentation
- Fixing UI/UX problems
- Optimizing performance
- Adding new NFT or metaverse-related functionality
- Improving accessibility
- Enhancing security
- Writing tests
- Reviewing pull requests

---

## Before You Start

Please take a moment to:

1. Read the project documentation.
2. Search existing Issues to avoid duplicates.
3. Open a discussion or issue before implementing major changes.
4. Follow the project's coding standards and architecture.

---

## Development Workflow

### 1. Fork the Repository

Create your own fork of the repository.

### 2. Clone Your Fork

```bash
git clone https://github.com/YOUR_USERNAME/Metaverse-Rang-Sovereign-Assets-NFT-VOD.git
cd Metaverse-Rang-Sovereign-Assets-NFT-VOD
```

### 3. Create a Branch

Use descriptive branch names.

```bash
git checkout -b feature/nft-gallery
```

Examples:

```text
feature/video-streaming
feature/metaverse-space
fix/mobile-navigation
fix/loading-performance
docs/readme-update
```

### 4. Make Your Changes

Keep commits focused and meaningful.

Good examples:

```text
feat: add NFT ownership verification
fix: resolve mobile menu overflow
docs: improve installation guide
refactor: optimize asset loading
```

### 5. Test Your Changes

Before submitting:

- Verify the project builds successfully.
- Ensure existing functionality still works.
- Test on multiple screen sizes when applicable.
- Run available tests.

### 6. Push and Open a Pull Request

```bash
git push origin feature/nft-gallery
```

Then create a Pull Request against the main branch.

---

## Pull Request Guidelines

When creating a Pull Request:

### Include

- Clear description of the change
- Motivation behind the change
- Screenshots (for UI updates)
- Related issue references

Example:

```text
Closes #42
```

### Keep Pull Requests

- Small and focused
- Easy to review
- Limited to one feature or fix

### Avoid

- Unrelated code changes
- Large formatting-only commits
- Mixing refactoring with feature additions

---

## Reporting Bugs

When reporting bugs, please include:

### Environment

- Browser
- Operating System
- Device type
- Screen size (if relevant)

### Description

- Expected behavior
- Actual behavior
- Steps to reproduce
- Screenshots or recordings

Example:

```text
1. Open homepage
2. Click NFT Gallery
3. Scroll to video section
4. Videos fail to load

Expected:
Videos should autoplay on visibility.

Actual:
Videos remain in loading state.
```

## Security

If you discover a security vulnerability, please do **not** open a public issue.

Instead, contact the maintainers privately and provide:

- Description of the vulnerability
- Potential impact
- Steps to reproduce
- Suggested mitigation

Responsible disclosure helps protect users and contributors.
